# OctoAcme — Project Planning

## Purpose
Turn an approved initiative into an actionable plan and backlog for delivery.

## Objectives
- Break work into shippable increments
- Identify dependencies and risks
- Align timelines, releases, and responsibilities
- **Establish clear role accountability and decision authority**

## Activities
1. Kickoff meeting with stakeholders and delivery team
2. Create prioritized backlog with acceptance criteria
3. **Clarify role assignments and responsibilities (see Role Clarity section)**
4. Estimate scope (T-shirt sizing or story points)
5. Define Definition of Done (DoD)
6. Identify dependencies and integration points
7. Create release plan and milestone map

## Role Clarity During Planning

### Before Kickoff
- [ ] Identify primary roles needed for this project (refer to `octoacme-roles-and-personas.md`)
- [ ] Assign individuals to each role
- [ ] Notify assigned team members of their role and responsibilities
- [ ] Ensure team members have read their role description

### During Kickoff
- [ ] Confirm role assignments and expectations
- [ ] Review role responsibilities and cross-functional interactions
- [ ] Clarify decision authority using the RACI matrix
- [ ] Establish role-specific communication preferences
- [ ] Identify gaps or overlaps in coverage
- [ ] Answer questions about role interactions

### Post-Kickoff
- [ ] Distribute role summary to all stakeholders
- [ ] Document any role-specific constraints or dependencies
- [ ] Confirm that team members understand their role in the project rhythm (standups, planning, retros)

## Sprint / Iteration Planning
- Timebox planning to agreed sprint length
- Pull items that meet DoD and have clear acceptance criteria
- **Confirm role ownership for each item** (e.g., Developer, QA Lead, Business Analyst)
- Ensure team capacity is respected

## Risk & Dependency Management
- Capture in Risk Register:
  - ID, Description, Impact, Probability, Owner **[with role]**, Mitigation
- Mark cross-team dependencies in the project board and escalate during weekly syncs
- **Identify role-specific dependencies** (e.g., Security Officer approval, Release Manager coordination)

## Planning Checklist
- [ ] Project kickoff held
- [ ] Role assignments clarified and communicated
- [ ] Backlog prioritized and estimated
- [ ] Release timeline and milestones agreed
- [ ] Definition of Done documented
- [ ] Initial test plan / QA approach drafted
- [ ] **Role-specific expectations and communication cadence confirmed**
- [ ] **RACI matrix reviewed and shared with team**
